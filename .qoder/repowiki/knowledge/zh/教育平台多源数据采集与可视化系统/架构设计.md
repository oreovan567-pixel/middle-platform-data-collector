- **分层架构**：采用经典的 MVC 变体结构。`web/` 层负责 HTTP 请求处理与页面渲染（Flask Blueprint）；`services/` 层为核心业务逻辑，特别是 `Collector` 负责多线程异步任务编排；`scrapers/` 层封装了针对不同平台（Grafana, Metabase, Main Site）的采集策略，支持 API 与 Playwright 浏览器模式的自动降级；`models/` 层基于 SQLite 实现数据持久化。
- **异步并发设计**：核心采集逻辑基于 `asyncio`，通过 `BrowserManager` 统一管理浏览器生命周期。`Collector` 利用 `asyncio.gather` 实现跨平台（如 Metabase 与主站）的并行采集，同时在单平台内对不同学校采用顺序执行以维持会话状态。
- **配置与凭证管理**：`config/config_loader.py` 提供统一的配置加载接口，支持 YAML 文件与环境变量覆盖。实现了用户级别的凭证覆盖机制，允许不同登录用户在采集时动态注入个人账号信息。
- **数据持久化与迁移**：`models/database.py` 负责 SQLite 数据库的初始化与 schema 迁移（如 `weekly_records`, `monthly_records` 表），支持在首次启动时从配置文件导入基础学校数据。

## 简介
本操作指南面向生产环境部署与运维，涵盖以下主题：
- 不同操作系统环境的部署要求（Python 环境、系统依赖、浏览器驱动）
- WSGI应用工厂模式和多种部署方式（本地、云平台、容器化、Serverless）
- Render.com和Vercel等云平台的零配置部署方案
- 性能调优参数（并发、线程、内存、数据库连接池）
- 监控告警与日志、健康检查接口
- 备份恢复、数据迁移与灾难恢复
- 故障排查、常见问题诊断与性能瓶颈分析
- 安全加固与访问控制、审计日志

## 项目结构
项目采用"Web 应用 + 采集服务 + 爬虫层 + 数据模型"的分层设计，核心目录与职责如下：
- config：配置加载与校验（YAML/环境变量）
- models：SQLite 数据模型与初始化、迁移
- services：采集编排器（Collector）、导出服务
- scrapers：爬虫层（浏览器自动化 Playwright + HTTP API）
- web：Flask Web 应用、路由与认证
- api：Vercel Serverless入口点
- data/logs：应用数据与日志目录
- 根目录：启动入口、WSGI应用工厂、云平台配置、依赖清单、批处理脚本

## 详细组件分析

### WSGI应用工厂模式
**更新** 项目采用标准的WSGI应用工厂模式，提供统一的应用创建接口和多平台部署支持：

- **应用工厂函数**
  - `create_app()` 函数集中管理应用初始化
  - 自动设置日志系统、模板路径、静态资源路径
  - 注册所有路由蓝图和认证中间件
  - 支持环境变量配置SECRET_KEY
- **多入口点支持**
  - `wsgi.py` 提供标准WSGI应用对象，兼容waitress、gunicorn
  - `api/index.py` 提供Vercel Serverless应用对象
  - 两个入口点都调用相同的create_app()函数
  - 自动处理Python路径和模块导入
- **多环境支持**
  - 开发环境：直接调用create_app()使用Flask开发服务器
  - 生产环境：通过WSGI服务器托管应用
  - 云平台：Render.com自动识别wsgi:app入口
  - Serverless：Vercel自动识别api/index.py入口

### 启动与运行模式
- **开发模式**
  - 使用 Flask 内置服务器，支持热重载
  - 默认端口5001，避免AirPlay冲突
  - 适合本地开发与调试
- **生产模式**
  - 使用 waitress，多线程、稳定、Windows 原生支持
  - 默认端口5000，线程数8，超时120秒
  - 未安装 waitress 时回退到 Flask（不建议生产使用）
- **云平台模式**
  - Render.com自动检测wsgi:app入口
  - 自动生成SECRET_KEY环境变量
  - 自动安装依赖和配置Python版本
- **Serverless模式**
  - Vercel自动检测api/index.py入口
  - 按需扩展，无需维护服务器
  - 自动HTTPS和全球CDN加速

### 云平台部署策略对比
**新增** 不同云平台的部署特点对比：

| 特性 | Render.com | Vercel |
|------|------------|---------|
| 部署类型 | 传统Web服务 | Serverless函数 |
| 启动时间 | 冷启动约30秒 | 冷启动约1-3秒 |
| 扩展性 | 手动或自动扩缩容 | 自动无限扩展 |
| 成本模型 | 按月计费 | 按使用量计费 |
| 持久化存储 | 支持卷挂载 | 需要外部存储 |
| 浏览器支持 | 完整Playwright支持 | 有限制的环境 |
| 适用场景 | 长期运行的服务 | 高并发短请求 |

**部署建议**
- **Render.com**：适合需要长时间运行、大量计算的任务
- **Vercel**：适合高并发、短请求的API服务
- **混合部署**：关键业务用Render，前端展示用Vercel

## 性能考虑
- **并发与线程**
  - waitress 线程数：默认 8，可通过 serve(...) 参数调整
  - 采集线程：Collector 使用后台线程执行任务
  - Vercel Serverless：自动水平扩展，无并发限制
- **浏览器资源**

  - 无头模式建议设置 viewport，避免页面渲染异常

## 故障排查指南
- **启动与运行**
  - waitress 未安装：回退到 Flask，建议安装 waitress 以获得稳定生产环境
  - 端口占用：确认 5000 端口可用或修改启动参数
  - WSGI导入错误：检查wsgi.py路径和模块导入
  - Vercel导入错误：检查api/index.py路径和sys.path配置
- **认证与会话**
  - 未登录访问 /api/* 返回 401：检查登录流程与会话写入
  - 默认管理员账户：admin/admin123（首次启动创建）
  - SECRET_KEY配置错误：检查环境变量设置
- **云平台部署**
  - Render.com构建失败：检查requirements.txt依赖版本
  - 环境变量缺失：确认SECRET_KEY已自动生成
  - 端口绑定错误：使用$PORT环境变量而非硬编码
  - Vercel部署失败：检查vercel.json配置和Python版本
  - Serverless内存不足：增加maxLambdaSize或优化代码
- **日志系统**
  - logs目录权限问题：确保应用有写入权限
  - 日志文件过大：配置日志轮转策略
  - 中文乱码：检查UTF-8编码设置
  - 云平台日志不可见：检查stdout/stderr输出
- **爬虫与浏览器**
  - 登录失败：检查 credentials 配置（用户名/密码/API Token）
  - Grafana 登录检测：多策略（CSS/URL/表单）确认登录状态
  - 主站一键登录：确认 Cloud 会话与 KS Cookie 获取流程
  - Serverless浏览器限制：考虑使用外部浏览器服务
- **数据库**
  - 表结构异常：检查迁移逻辑与 WAL/外键设置
  - 首次导入学校：确认 config.yaml 存在且格式正确
  - Serverless数据库连接：使用外部数据库服务
- **API 采集**
  - Metabase/Grafana/主站 API 返回空：检查凭据、时间范围与筛选参数
  - 401 会话失效：清理缓存 Cookie，重新获取会话与 Cookie
